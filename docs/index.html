Quick Start

Run with npx — no global install required.

Scaffold a new project

npx create-spec-driven-app@latest init \
  --config ./project.config \
  --out ./projects

Validate generated project

create-spec-driven-app validate \
  ./projects/my-app

Expand a domain pack

create-spec-driven-app expand \
  --pack packs/auth/backend/pack.yaml \
  --out ./projects/my-app

Scaffold a new domain pack

create-spec-driven-app pack init \
  --name payments \
  --out ./packs

Lint a domain pack

create-spec-driven-app pack lint \
  packs/auth/backend/pack.yaml

Install globally

npm i -g create-spec-driven-app

Why teams use this

Spec-first delivery

Define requirements before implementation. Traceability matrix links REQ-NNN IDs from spec to tests to source.

10 domain packs

Auth, billing, notifications, audit-log, feature-flags, multi-tenant, file-storage, search, reporting, and webhooks — ready to expand.

AI-native

MCP server exposes CLI tools to any MCP-compatible AI agent. VS Code extension adds schema validation and traceability navigation.

BDD + TDD built-in

Gherkin feature files and Cucumber step definitions scaffolded alongside the project. 178 tests across unit, BDD, and property-based layers.

Pure Node.js

No Bash dependency. Works on Linux, macOS, and Windows (Node 20+). All scripts ported to cross-platform JS.

CI/CD ready

Preconfigured GitHub Actions for tests (Linux/macOS/Windows matrix), npm publish, GitHub Packages, and GitHub Pages registry.

Domain Packs 10 packs

Each pack ships a pack.yaml with domain model, requirements, Gherkin scenarios, API contracts, and template stubs. Use expand to inject them into any generated project.

Ecosystem

VS Code Extension New

Schema squigglies for pack.yaml, hover docs for domain fields, and a Reveal in Traceability Matrix command that jumps from any REQ-NNN identifier to its row in the matrix.

Source Download .vsix

MCP Server AI-native

JSON-RPC server that exposes five tools to any MCP-compatible AI agent: init_project, validate_project, expand_pack, list_packs, lint_pack. Works with Claude, Cursor, Copilot, and more.

Source npm

Pack Registry

Static-site generator that builds a searchable HTML catalog and a machine-readable manifest.json from all packs in the repository. Self-hosted on GitHub Pages.

Browse registry manifest.json Source

Docker / Dev Container New

Multi-environment .env.* files, Docker Compose services, and a .devcontainer/devcontainer.json scaffold generated by the runtime-environments template.

Deployment docs

Command Reference

Command Description
init Generate a new SDD project from templates.
validate Validate required structure, files, and unresolved placeholders.
expand Inject a domain pack (pack.yaml) into an existing project.
pack init Scaffold a new pack.yaml with all required fields.
pack lint Validate a pack.yaml against the JSON Schema contract.
--dry-run Preview actions without writing files.
--no-git Skip git init in generated output.
--force Overwrite destination when it already exists.
