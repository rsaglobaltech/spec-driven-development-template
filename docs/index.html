The problem

Spec and implementation drift apart the moment you start coding.

Requirements live in Confluence. Tests live in Jest. Domain models live in developers' heads. There is no single source of truth linking business intent to running code — so regressions slip through, and audits become archaeology.

📋

Spec-Driven Development

Every project gets a machine-readable spec.md with REQ-NNN identifiers, a traceability matrix, and schema-validated domain packs — all generated, not hand-maintained.

🗺️

Domain-Driven Design

Bounded contexts, aggregates, domain events, and a ubiquitous-language glossary scaffold out of the box, keeping your domain model explicit and evolvable.

🥒

Behavior-Driven Development

Gherkin .feature files and Cucumber step definitions are generated alongside source code, so acceptance criteria are executable from commit one.

🧪

Test-Driven Development

Unit, property-based, and BDD test scaffolds generated per command. 178 tests, mutation-tested baseline, and a coverage gate of ≥ 80 % line coverage.

How it works

Three commands to a spec-first project.

Each step produces concrete, linkable artifacts — not boilerplate that gets deleted on day two.

01 init

Scaffold the project

Generates spec.md, traceability matrix, Gherkin feature files, directory structure, Docker Compose, dev-container, and .env.* per environment.

npx create-spec-driven-app init \
  --config ./project.config \
  --out ./my-app
02 expand

Inject a domain pack

Merges a curated domain pack — requirements, aggregates, events, scenarios — into the project. Each pack is JSON Schema–validated before expansion.

npx create-spec-driven-app expand \
  --pack-root ./packs \
  --pack auth/backend \
  --project-dir ./my-app
03 validate

Enforce the contract

Checks directory structure, required files, unresolved template placeholders, and spec linkage. Run in CI to gate every PR.

npx create-spec-driven-app validate \
  ./my-app

CLI Reference

All commands at a glance.

Scaffold a project

npx create-spec-driven-app init \
  --config ./project.config \
  --out ./projects

Validate a project

create-spec-driven-app validate \
  ./projects/my-app

Expand a domain pack

create-spec-driven-app expand \
  --pack-root ./packs \
  --pack auth/backend \
  --project-dir ./my-app

Create a new pack

create-spec-driven-app pack init \
  --name payments \
  --out ./packs

Lint a pack

create-spec-driven-app pack lint \
  packs/auth/backend/pack.yaml

Install globally

npm install -g create-spec-driven-app
FlagDescription
--dry-runPreview actions without writing files.
