Quick Start

Install globally or run directly with npx.

Run with npx

npx create-spec-driven-app@latest init \
  --config ./project.config \
  --out ./projects

Install globally

npm i -g create-spec-driven-app

Validate generated project

create-spec-driven-app validate ./projects/my-app

Why teams use this

Spec-first delivery

Define requirements before implementation with production-grade defaults.

Traceability built-in

Keep a clear path from business intent to scenarios and technical artifacts.

Reusable templates

Stay domain-agnostic by default and add modules only when needed.

CI/CD friendly

Preconfigured workflows for tests, packaging checks, and npm publishing.

Command Reference

Command Description
init Generate a new SDD project from templates.
validate Validate required structure, files, and unresolved placeholders.
--dry-run Preview actions without writing files.
--no-git Skip git init in generated output.
--force Overwrite destination when it already exists.

Publish workflows

Two release lanes are supported from GitHub Actions:

  • npm registry: publish-npm.yml
  • GitHub Packages: publish-github-packages.yml
